# Hindsight — UI Design Spec

## Overview

Trade-centric day trading journal. Dark theme with green/red P&L semantics. Desktop-first (used during US market hours at a desk). Personal tool and portfolio project.

## Design Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Mental model | Trade-Centric | Most proven pattern for trading journals. Home = trade list, drill into details. |
| Navigation | Collapsible sidebar | Desktop-first app with 5 nav items. Icons when collapsed for more chart space. |
| Color palette | Dark + Green/Red | Trading-native. No accent color. P&L colors (#22C55E / #EF4444) dominate. |
| Dashboard | Balanced split | Left: metrics + equity curve. Right: today's trades. Both visible without scrolling. |
| Trade detail | Full detail | All data on one page: prices, P&L, R-multiple, grades, notes, screenshots. |
| Analytics | Full dashboard | Equity curve, strategy win rate, day-of-week P&L, hourly heatmap, R-distribution, streaks, drawdown. |
| Sessions | Full guided review | Market condition, mood/energy (1-5), pre-market plan, daily grade (A-F), post-market scorecard, auto-stats, review notes. |

## Navigation

```
Collapsible Sidebar
├── Dashboard (home)
├── Trades
├── Analytics
├── Sessions
└── Settings
```

Collapsed state shows icons only (48px wide). Expanded shows icon + label (~200px). User toggles via button at top of sidebar.

## Color System

### Base Palette

| Token | Value | Usage |
|---|---|---|
| `--bg-deep` | `#080808` | Sidebar background |
| `--bg-base` | `#0c0c0c` | Page background |
| `--bg-elevated` | `#111111` | Main content area, cards |
| `--bg-surface` | `rgba(255,255,255,0.04)` | Nested surfaces |
| `--border` | `rgba(255,255,255,0.08)` | Card borders, dividers |
| `--border-subtle` | `rgba(255,255,255,0.04)` | Inner dividers |
| `--foreground` | `#e5e5e5` | Primary text |
| `--foreground-muted` | `#666666` | Secondary text, labels |
| `--foreground-dim` | `#444444` | Placeholder text |

### Semantic Colors

| Token | Value | Usage |
|---|---|---|
| `--profit` | `#22C55E` | Winning trades, positive P&L, gains |
| `--loss` | `#EF4444` | Losing trades, negative P&L, drawdown |
| `--profit-bg` | `rgba(34,197,94,0.03)` | Win row highlight |
| `--loss-bg` | `rgba(239,68,68,0.03)` | Loss row highlight |
| `--profit-surface` | `rgba(34,197,94,0.15)` | Win badges, tags |
| `--loss-surface` | `rgba(239,68,68,0.15)` | Loss badges, tags |

### No Accent Color

This palette intentionally has no accent color. Green and red carry all semantic meaning. Neutral whites/grays handle all UI chrome (active nav, buttons, focus rings). This keeps the interface focused on P&L data.

## Typography

- Font family: System font stack (`-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`)
- Monospace: `ui-monospace, 'Cascadia Code', 'Fira Code', monospace` for prices, P&L, R-multiples
- Base size: 16px
- Line height: 1.5 body, 1.3 headings

| Role | Size | Weight | Color |
|---|---|---|---|
| Page title | 14px | 600 | `--foreground` |
| Section label | 8px (uppercase) | 400 | `--foreground-muted` |
| Metric value | 15-18px | 700 | varies by context |
| Metric label | 8px (uppercase) | 400 | `--foreground-muted` |
| Table header | 9px | 400 | `--foreground-muted` |
| Table cell | 9px | 400-600 | varies |
| Body text | 10px | 400 | `#999` |

## Component Patterns

### Cards

```css
background: rgba(255, 255, 255, 0.04);
border: 1px solid rgba(255, 255, 255, 0.08);
border-radius: 8px;
padding: 10px;
```

### Sidebar

- Background: `--bg-deep`
- Icon size: 32x32px, border-radius 6px
- Active state: `background: rgba(255,255,255,0.1); color: #fff`
- Inactive state: `background: rgba(255,255,255,0.05); color: #666`
- Settings/overflow at bottom

### Metric Cards

Grid layout (responsive columns). Each card has:
- Uppercase label (8px, muted)
- Value (15px, bold, contextual color)
- Subtitle (9px, muted)

### Trade Rows

Inline P&L coloring on the row background (`--profit-bg` or `--loss-bg`). Ticker in bold. P&L and R-multiple use semantic colors.

### Badges/Tags

```css
/* Profit badge */
color: #22C55E;
background: rgba(34,197,94,0.15);
padding: 2px 8px;
border-radius: 4px;
```

### Grade Indicators

Conviction and process grades (A/B/C) displayed in small badge with semantic coloring:
- A: green badge
- B: muted badge
- C: red badge

### Toggle Button Groups

For market condition, yes/no scorecard items. Active item gets colored border + fill, inactive gets subtle background.

### Rating Scales (Mood/Energy)

1-5 number boxes. Selected box gets green border + fill. Unselected get subtle background.

### Charts

Bar charts use green/red fills with varying opacity. No accent color. Grid lines at `rgba(255,255,255,0.06)`. Axis labels in `--foreground-muted`.

## Page Layouts

### Dashboard

```
┌─────────────────────────────────────────────┐
│ [Sidebar 48px] │  Main Content Area         │
│                │  ┌─────────┬───────────┐    │
│  [D]           │  │ Metrics │  Today's  │    │
│  [T]           │  │ +       │  Trades   │    │
│  [A]           │  │ Equity  │  Table    │    │
│  [S]           │  │ Curve   │           │    │
│  [?]           │  └─────────┴───────────┘    │
└─────────────────────────────────────────────┘
```

Left panel (~45%): 2x2 metric grid + equity curve chart
Right panel (~55%): Today's trades table (ticker, side, P&L, R, time)

### Trade Detail

Full-page scrollable layout:
1. Header: ticker + side badge + strategy + date range + net P&L
2. Price grid: entry, exit, quantity (3-column)
3. Risk grid: stop, target, R-multiple, commission (4-column)
4. Grades: conviction (A/B/C), process (A/B/C), net P&L (3-column)
5. Notes: text block
6. Screenshots: entry + exit image uploads side-by-side

### Analytics

Full dashboard with date range picker (7D / 30D / All):
1. Top row: 4 summary metric cards (win rate, profit factor, avg winner, avg loser)
2. Equity curve (full width, tallest chart)
3. Two-column: hourly heatmap | R-multiple distribution
4. Two-column: streak tracker | drawdown chart
5. Two-column: win rate by strategy | P&L by day of week

### Sessions

Full guided review per trading day:
1. Header: date + session hours + auto-populated daily P&L + trade count
2. Status row: market condition (tag select), mood (1-5), energy (1-5), daily grade (A-F)
3. Pre-market plan: text area
4. Post-market scorecard: checklist (followed risk rules, waited for setups, no forced trades, hit daily target)
5. Auto stats row: P&L, win rate, avg R, commission (pulled from day's trades)
6. Review notes: text area

### Settings

IBKR connection configuration, preferences, backup settings. Standard form layout.

## Spacing Scale

4px increments: 4, 8, 12, 16, 20, 24, 32, 48

## Border Radius

- Cards/panels: 8px
- Buttons/badges: 4px
- Sidebar icons: 6px
- Chart containers: 8px

## Responsive Behavior

Desktop-first app. Minimum target: 1024px viewport. No mobile-specific layout in v1. Sidebar collapses to icons below 1280px.
